0
🔑 前提條件 PREREQUISITE
攻擊者持有一組低權限域帳號（非 Domain Admin）。
此帳號僅用於觸發 PetitPotam，無法直接存取高價值資源。
AD CS 必須啟用 Web Enrollment，且未啟用 EPA（Extended Protection for Authentication）。
攻擊者 持有 lowpriv@domain + AD CS Web Enrollment 開啟
1
📡 PetitPotam：強制 DC 發起 NTLM 認證 TRIGGER
攻擊者用低權限帳號呼叫 DC 的 EFS RPC (MS-EFSRPC) 介面，
強制 DC 以 DC 機器帳號（DC$） 身分向攻擊機發起 NTLM 認證。
此時 ntlmrelayx 已在攻擊機監聽 Port 445。
攻擊者 ──EFS RPC──▶ DC ──NTLM Auth──▶ 攻擊機:445
2
🔀 ntlmrelayx：同步轉發 NTLM 至 AD CS RELAY（同步）
ntlmrelayx 即時將 DC 的 NTLM Challenge/Response 轉發至 AD CS HTTP Endpoint。
AD CS 驗證通過（誤以為請求來自 DC 本身），允許以 DC$ 身分申請憑證。
注意：這是同步操作，ntlmrelayx 作為 MITM 橋接兩端，不是先截獲再轉發。
DC ──NTLM──▶ ntlmrelayx (MITM) ──同步轉發──▶ AD CS HTTP
3
📜 AD CS 核發 DC 機器帳號憑證 CERT ISSUED
AD CS 核發一張以 DC$ 為主體的憑證（.pfx / Base64）。
ntlmrelayx 直接在攻擊機本地儲存此憑證，CA 不會主動推送。
Event Log：CA 產生 4886（收到申請）、4887（憑證核發），Requester 顯示為 DOMAIN\DC$。
AD CS ──核發憑證──▶ ntlmrelayx 儲存 → dc.pfx (DC$ 憑證)
4
🎫 certipy auth：PKINIT 取得 DC$ 的 TGT PKINIT
使用 DC$ 憑證透過 PKINIT（憑證式 Kerberos） 向 KDC 申請 TGT。
同時可取得 DC$ 的 NT Hash（透過 U2U 機制）。
Event Log：DC 產生 4768，PreAuthType = 16（非一般密碼登入的 2，這是異常特徵）。
dc.pfx ──certipy auth──▶ KDC (DC) ──TGT──▶ DC$.ccache + NT Hash
5
🎭 getST.py S4U2Self：模擬 Administrator PRIVILEGE ESC
DC$ 機器帳號天生具有 TrustedToAuthForDelegation 屬性，
允許透過 S4U2Self 以自身身分代替任意使用者（如 Administrator）申請 Service Ticket。
Event Log：DC 產生 4769，TransitedServices 不為空，AccountName 為機器帳號。
DC$.ccache ──S4U2Self──▶ KDC ──▶ ST: Administrator@cifs/DC
6
👑 Domain Admin 完整控制 PWNED
使用 Administrator 票證執行 wmiexec / secretsdump (DCSync)，
取得 krbtgt NT Hash，可製作 Golden Ticket，永久維持域控制權。
Event Log：4624 Administrator 登入、4672 特權指派、4662 含複寫 GUID（DCSync）。
Administrator.ccache ──▶ DCSync ──▶ krbtgt Hash → Golden Ticket
⚡ 核心概念修正：
攻擊者需要一組低權限域帳號來觸發 PetitPotam，但不需要任何高權限帳號的密碼。
ntlmrelayx 執行的是同步 MITM 轉發，不是先截獲再重放——這也是為何 NTLM Relay 無法被 NTLMv2 本身防禦，必須靠 SMB Signing 或 EPA。
🛡 關鍵防禦措施：
• 啟用 SMB Signing（防止 SMB Relay）
• AD CS 啟用 EPA（Extended Protection for Authentication）（防止 HTTP Relay）
• 啟用 LDAP Signing + Channel Binding
• 封鎖非 DC 主機對 DC 的 EFS RPC 呼叫（MS-EFSRPC）
